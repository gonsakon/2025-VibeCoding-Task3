個人網站 Hero 區占位內容

這裡是前端工程師個人網站的主視覺 Hero 區

目前這裡是用來示意版面結構的占位文字。未來會使用「Section SDD」的 Prompt， 產生包含一句話自我介紹、核心職稱（例如 Frontend Developer）、 以及清楚的行動呼籲（例如瀏覽作品集或下載履歷）的正式文案與排版。

你可以把這一區想像成訪客一進到網站就會看到的第一個畫面。之後產生的新 Hero 區程式碼， 會「整段」貼在 SECTION:HERO-START / SECTION:HERO-END 兩個註解中間， 完全取代目前這個占位內容。

示意按鈕：之後會放「瀏覽作品集」等行動呼籲 示意按鈕：之後可放「下載履歷」或其他 CTA

提醒開發者：之後會用 Section SDD 的 Prompt 產生新的 Hero section，請把新的 section 程式碼整段貼在這兩個註解中間，完全取代現在這一區。

示意圖片：未來可以替換為個人照片或更符合品牌風格的 Unsplash 圖片，這裡先提供一張與程式開發相關的占位圖。

關於我（之後會用 Section SDD 替換）

這裡是 About 區的占位內容，主要用來示意「介紹前端工程師本人」的版面結構。未來會透過 Section SDD（單一區塊規格），填寫更具體的背景、專長與工作型態描述。

例如：你可能會在正式版本中放上「目前職稱」、「工作年資」、「擅長的網頁領域（如 RWD、Landing Page、前端架構）」 以及「個人教學或專案經驗」。這些細節都會寫在 SDD 規格中，再由 AI 依照規格輸出完整 HTML + CSS。

之後會用 Section SDD 的 Prompt 產生新的 About section，請把新的 section 程式碼整段貼在 SECTION:ABOUT-START 與 SECTION:ABOUT-END 兩個註解中間，完全取代現在這一整區的內容。

區塊說明給開發者

建議在 About 的 SDD 中，清楚列出想呈現的「人物設定」與「目標受眾」，例如：轉職前端、自由接案、 或帶有教學角色的工程師。AI 會依照這些規格，產生更貼近個人品牌的敘事與排版。

小提醒：未來替換時，請直接複製新的 <section id="about" class="section section-about">…</section> 整段貼上，避免只局部修改造成結構不一致。

技能區（之後會用 Section SDD 替換）

這裡是 Skills 區的占位內容，用來示意將來會呈現「前端技術能力地圖」的位置。 未來會透過 Section SDD 明確指定要展示哪些技能分類、熟悉程度與說明文字。

你可以在 SDD 中規劃，例如：分成「前端核心」、「版型整合」、「開發流程」三大類，並列出 HTML、CSS、JavaScript、 Git、RWD、程式設計思維等項目。AI 會根據這些規格產出對應的清單或標籤式排版。

示意：技能標籤區

下方技能標籤純屬示意，讓你看出未來可以如何排版。實際顯示的技術與分類，請在 Section SDD 中詳細定義，再交給 AI 產出新程式碼。

  • HTML / CSS（基礎排版與語意化結構）
  • RWD 響應式設計
  • 設計稿切版流程
  • 版本控制（Git / GitHub）

之後會用 Section SDD 的 Prompt 產生新的 Skills section，請把新的 section 程式碼整段貼在 SECTION:SKILLS-START / SECTION:SKILLS-END 之間，完全取代目前這一區。

作品集區（之後會用 Section SDD 替換）

這裡是 Projects 區的占位內容，負責展示未來要放上的「個人專案」、「練習作品」或「商業合作案例」。 目前僅示意卡片式排版與圖片結構，實際專案名稱與說明將在 Section SDD 中決定。

作品卡片占位示意（一）

未來可以在這裡放上「個人網站」、「Landing Page」或「Side Project」等專案名稱與簡短說明。 目前文字僅用來示意排版位置與層級結構。

提醒開發者：真正的專案內容會由 Section SDD 定義（例如目標族群、使用技術、負責範圍），再請 AI 輸出新的 section 程式碼來覆蓋。

作品卡片占位示意（二）

你可以在 SDD 中規劃是否需要「展示圖片」、「Demo 連結」或「GitHub 連結」等欄位。 這裡先以純文字描述示意，方便之後整段替換。

之後會用 Section SDD 的 Prompt 產生新的 Projects section，請把新的 section 程式碼整段貼在 SECTION:PROJECTS-START / SECTION:PROJECTS-END 兩個註解中間，讓作品集區完整更新。

聯絡我（之後會用 Section SDD 替換）

Contact 區會是整個個人網站對外開放聯絡方式的地方。未來可以在 Section SDD 中決定要呈現的管道，例如 Email、 手機、GitHub、LinkedIn 或其他社群平台，再由 AI 依據規格生成排版與文字。

  • Email： hello@example.com
  • 電話： +886 900 000 000
  • GitHub： https://github.com/yourname
  • LinkedIn： https://www.linkedin.com/in/yourname

之後會用 Section SDD 的 Prompt 產生新的 Contact section，請把新的 section 程式碼整段貼在 SECTION:CONTACT-START 與 SECTION:CONTACT-END 兩個註解中間，完全取代目前這一區，並確保所有 href 仍然是有效且非空的連結。
